# LeetCode #3043 - 最长公共前缀的长度

- **难度**: Medium
- **日期**: 2026-05-26 AC
- **语言**: TypeScript
- **Runtime**: 187ms | Memory: 85.9 MB

## 题意

给两个正整数数组，找所有跨数组配对中最长公共前缀（数字从左到右的连续相同位）的长度。

## 思路

1. 用 arr1 的所有数字按位构建一棵 Trie（字典树），节点 children 用 Map<digit, Node>
2. 遍历 arr2 的每个数字，在 Trie 中逐位查询，能走多深就是该对的最长公共前缀长度
3. 取所有查询结果的 max

## 复杂度

- 时间: O((n₁ + n₂) × L)，L ≤ 9（数字最多 9 位）
- 空间: O(n₁ × L) Trie 节点数

## 代码

```typescript
function longestCommonPrefix(arr1: number[], arr2: number[]): number {
    const root: Map<number, any> = new Map();
    for (const num of arr1) {
        const digits = String(num);
        let node = root;
        for (const ch of digits) {
            const d = Number(ch);
            if (!node.has(d)) {
                node.set(d, new Map());
            }
            node = node.get(d)!;
        }
    }
    let ans = 0;
    for (const num of arr2) {
        const digits = String(num);
        let node = root;
        let depth = 0;
        for (const ch of digits) {
            const d = Number(ch);
            if (!node.has(d)) break;
            node = node.get(d)!;
            depth++;
        }
        ans = Math.max(ans, depth);
    }
    return ans;
}
```

## 关键洞察

- 这题天然适合 Trie：本质是"所有前缀的集合匹配"，Trie 可以一次性存储所有数字的所有前缀
- Map vs Array(10)：digits 只有 0-9，用定长数组理论上更快（O(1) 下标访问 vs Map 的 hash），但 Map 写起来更简洁，两者都能过
- 数字转字符串逐位处理比反复取模更直观，且避免了反转的麻烦
- 也可以用 HashSet 存 arr1 所有前缀（123→{1,12,123}），再用 arr2 的前缀去查，但空间和常数都不如 Trie 优雅
