# 3742. Maximum Path Score in a Grid（网格中得分最大的路径）

- **难度**: Medium
- **标签**: Array, DP, Matrix
- **日期**: 2026-05-17
- **耗时**: 366ms | 66.5MB

## 思路

带预算约束的网格 DP。`dp[i][j][c]` = 到 (i,j) 恰好花费 c 的最大得分。

### 关键优化：滚动数组

- 依赖只有**上方**和**左方**，可以压掉行维度 → `dp[j][c]`
- c 从大到小遍历（类似 0-1 背包），确保读到的 `dp[j][c-cost]` 是上一行的旧值
- 空间从 O(m×n×k) 降到 O(n×k)

### 格子规则

| 值 | 得分 | 花费 |
|----|------|------|
| 0  | 0    | 0    |
| 1  | 1    | 1    |
| 2  | 2    | 1    |

## 代码

```typescript
function maxPathScore(grid: number[][], k: number): number {
    const m = grid.length;
    const n = grid[0].length;

    const dp: number[][] = Array.from({ length: n }, () => new Array(k + 1).fill(-1));
    dp[0][0] = 0;

    for (let i = 0; i < m; i++) {
        for (let j = 0; j < n; j++) {
            if (i === 0 && j === 0) continue;

            const cost = grid[i][j] === 0 ? 0 : 1;
            const score = grid[i][j];

            for (let c = k; c >= cost; c--) {
                const above = dp[j][c - cost];
                const left = j > 0 ? dp[j - 1][c - cost] : -1;

                let best = -1;
                if (above > best) best = above;
                if (left > best) best = left;

                dp[j][c] = best === -1 ? -1 : best + score;
            }

            for (let c = 0; c < cost; c++) {
                dp[j][c] = -1;
            }
        }
    }

    let ans = -1;
    for (let c = 0; c <= k; c++) {
        if (dp[n - 1][c] > ans) ans = dp[n - 1][c];
    }
    return ans;
}
```

## 复杂度

- 时间：O(m × n × k)
- 空间：O(n × k)（滚动数组优化）

## 踩坑记录

- 初版只存 [k剩余, 分数] 一个对——反例证明会丢失"分低但预算充裕"的路径
- 滚动数组中 c 必须从大到小遍历，否则读到的是当前行新值而非上一行旧值（0-1背包经典）
