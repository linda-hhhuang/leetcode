# LeetCode #3620 - 网络恢复路径

- **难度**: Hard
- **日期**: 2026-07-14 AC
- **语言**: TypeScript
- **Runtime**: 1482ms | Memory: 127.7 MB

## 题意

DAG 有离线节点，从 0 到 n-1 找合法路径（中间节点全在线、总 cost ≤ k），最大化路径上的最小边权（score）。

## 思路

1. 预处理：去掉离线节点的边 + cost > k 的边
2. 收集所有边权值去重排序作为二分候选
3. 二分 mid：只保留 cost ≥ mid 的边 → 拓扑排序 + 松弛求 DAG 最短路 → 判 dist[n-1] ≤ k
4. 能通则 lo = mid（score 至少是 mid），不能通则 hi = mid - 1

## 复杂度

- 时间: O((n+m) × log(m))，二分 log(m) 轮，每轮拓扑+松弛 O(n+m)
- 空间: O(n+m)

## 代码

```typescript
function findMaxPathScore(edges: number[][], online: boolean[], k: number): number {
    const n = online.length;
    const validEdges: [number, number, number][] = [];
    for (const [u, v, cost] of edges) {
        if (!online[u] || !online[v]) continue;
        if (cost > k) continue;
        validEdges.push([u, v, cost]);
    }
    const costs = [...new Set(validEdges.map(e => e[2]))].sort((a, b) => a - b);
    if (costs.length === 0) return -1;

    function canAchieve(mid: number): boolean {
        const adj: [number, number][][] = Array.from({length: n}, () => []);
        const inDeg = new Array(n).fill(0);
        for (const [u, v, cost] of validEdges) {
            if (cost >= mid) { adj[u].push([v, cost]); inDeg[v]++; }
        }
        const queue: number[] = [];
        for (let i = 0; i < n; i++) if (inDeg[i] === 0) queue.push(i);
        const order: number[] = [];
        let qi = 0;
        while (qi < queue.length) {
            const u = queue[qi++]; order.push(u);
            for (const [v] of adj[u]) { if (--inDeg[v] === 0) queue.push(v); }
        }
        const dist = new Array(n).fill(Infinity);
        dist[0] = 0;
        for (const u of order) {
            if (dist[u] === Infinity) continue;
            for (const [v, w] of adj[u]) {
                if (dist[u] + w < dist[v]) dist[v] = dist[u] + w;
            }
        }
        return dist[n - 1] <= k;
    }

    let lo = 0, hi = costs.length - 1;
    if (!canAchieve(costs[0])) return -1;
    while (lo < hi) {
        const mid = (lo + hi + 1) >> 1;
        if (canAchieve(costs[mid])) lo = mid;
        else hi = mid - 1;
    }
    return costs[lo];
}
```

## 关键洞察

- 跟 #2812 同一个模式："最大化路径最小值" = 二分答案
- DAG 最短路用拓扑排序+松弛 O(n+m)，比 Dijkstra 少 log 因子
- 预先去 cost > k 的边（因为 cost 非负，单边 > k 的路径必超限）是合理剪枝
- 函数名要看 LeetCode 的函数签名（这次是 `findMaxPathScore`），不能自己起名
