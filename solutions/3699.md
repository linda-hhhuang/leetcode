# LeetCode #3699 - 锯齿形数组的总数 I

- **难度**: Hard
- **日期**: 2026-06-29 AC
- **语言**: TypeScript
- **Runtime**: 563ms (number 版) / 755ms (BigInt 版) | Memory: 62.3 MB

## 题意

求长度 n、值域 [l,r]、相邻不等、任意连续三个不严格单调（即每个中间元素是局部极大或极小）的数组总数，mod 1e9+7。

## 思路

DP 双状态 + 前缀和优化，再用对称性砍半：

1. 值域映射到 0..k-1（k=r-l+1）
2. 定义 `up[v]`=以 v 结尾且最后一步是上升的方案数，`down[v]`=最后一步下降
3. 转移：
   - `newUp[w] = ∑_{v<w} down[v]`（前缀和）
   - `newDown[w] = ∑_{v>w} up[v]`（后缀和）
4. **对称性**：值域翻转 x→l+r-x 保持 ZigZag 性质且 up↔down 互换，故 `up[v] = down[k-1-v]`
5. 只维护 down 一个数组，`newDown[w] = ∑_{v>w} down[k-1-v] = prefix[k-1-w]`
6. 答案 = 2·∑down[v]

## 复杂度

- 时间: O(n×k)
- 空间: O(k)

## 代码

```typescript
function zigZagArrays(n: number, l: number, r: number): number {
    const MOD = 1000000007;
    const k = r - l + 1;
    let down = new Array(k).fill(1);
    for (let step = 2; step <= n; step++) {
        const prefix = new Array(k + 1).fill(0);
        for (let i = 0; i < k; i++) prefix[i + 1] = (prefix[i] + down[i]) % MOD;
        const nd = new Array(k).fill(0);
        for (let w = 0; w < k; w++) nd[w] = prefix[k - 1 - w];
        down = nd;
    }
    let half = 0;
    for (let v = 0; v < k; v++) half = (half + down[v]) % MOD;
    return (half * 2) % MOD;
}
```

## 关键洞察

- 状态只需记"方向 + 当前值"，因为 ZigZag 的下一步合法性只取决于上一步是升还是降
- 双重求和用前缀/后缀和摊成 O(1)，整体降到 O(n×k) 卡进 n,k≤2000
- 对称性 up[v]=down[k-1-v] 来自值域翻转不变性，省一半计算和空间
- 手算验证时易漏迭代次数：n=3 需要 step=2..3 共 2 步，不是 1 步（踩坑点）
- **number vs BigInt**：本题只有加法和最后乘 2，无大数相乘。每次 %MOD 后值 < 1e9+7，两数相加 < 2e9 < Number.MAX_SAFE_INTEGER(9e15)，乘 2 也 < 2e9，所以 number 完全安全且快约 2x（563ms vs 755ms）。BigInt 仅在需要"两个 mod 级别数相乘"（积可达 1e18 > MAX_SAFE）时才必需
